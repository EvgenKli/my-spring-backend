# Production-Ready Java Spring Boot REST API Backend

Высокотехнологичный облачный микросервис на базе фреймворка Spring Boot (v3.2.4), функционирующий в рамках гео-распределенного промышленного кластера Kubernetes в облаке Selectel (OpenStack API).

## Архитектура распределенной системы
Приложение спроектировано по классической трехслойной корпоративной архитектуре (Controller-Repository-Entity) и работает в связке с удаленными инфраструктурными компонентами:

*   **Фронтенд-дверь:** Веб-контроллер (`UserController.java`) обслуживает сетевые REST-маршруты по пути `/users`.
*   **Слой данных (ORM):** Спринговый репозиторий (`UserRepository.java`) на базе Spring Data JPA/Hibernate автоматически транслирует объекты Java в реляционные сущности базы данных.
*   **Облачная СУБД:** Данные по сети улетают в Москву на выделенный сервер СУБД **PostgreSQL 16**, развернутый и настроенный с нуля с помощью Ansible.

## Контейнеризация и Оптимизация (Docker)
Для изоляции приложения разработан профессиональный многоэтапный **Multi-stage Dockerfile**:
1.  **Stage 1 (Builder):** На базе тяжелого образа `maven:3.9.6-eclipse-temurin-21-alpine` происходит контролируемая компиляция исходного кода (`src/`) и сборка Fat-JAR артефакта.
2.  **Stage 2 (Final):** В легковесный продакшен-образ `eclipse-temurin:21-jre-alpine` переносится исключительно скомпилированный `.jar` файл. 
*   **Результат:** Размер итогового Docker-образа сокращен **в 10 раз** (с 500 МБ до 50 МБ), что минимизирует зону атаки (Attack Surface) и ускоряет деплой.

## Управление в кластере Kubernetes
Микросервис успешно развернут внутри Bare-Metal кластера Kubernetes (развернут с помощью `kubeadm`). 
*   **Отказоустойчивость:** Архитектура `Deployment` автоматически поддерживает в актуальном состоянии **2 реплики (Пода)** приложения на воркер-ноде.
*   **Баласировка:** Внутренний трафик кластера распределяется между подами с помощью сетевого абстрактного компонента `Service` (ClusterIP).

## Спецификация REST API запросов

### Создание пользователя (POST)
```bash
curl -X POST http://localhost:8080/users \
  -H "Content-Type: application/json" \
  -d '{"name": "EvgenKli", "email": "devops@example.com"}'
```

### Получение списка пользователей (GET)
```bash
curl http://localhost:8080/users
```
