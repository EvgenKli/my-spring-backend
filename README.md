# Cloud-Native Java Spring Boot REST API Backend with GitLab CI/CD

Промышленный бэкенд-микросервис на базе фреймворка Spring Boot (v3.2.4), функционирующий в распределенном облачном кластере Kubernetes под управлением автоматического конвейера сборки.

## Архитектура и функционал приложения
Приложение реализует REST API управления сущностями пользователей и построено по слоям (Controller -> Repository -> Entity):
*   `POST /users` — Прием JSON-данных, валидация и запись через Hibernate ORM в удаленную облачную СУБД PostgreSQL.
*   `GET /users` — Высокоскоростное чтение списка пользователей из базы данных.

## Контейнеризация (Dockerfile)
Спроектирован оптимизированный многоэтапный **Multi-stage Dockerfile**:
*   **Этап 1 (Builder):** Компиляция кода утилитой Maven на базе тяжелого образа.
*   **Этап 2 (Final JRE):** Копирование готового бинарного `.jar` файла в ультра-легкий дистрибутив `eclipse-temurin:21-jre-alpine`.
*   **Результат:** Размер продакшен-образа уменьшен **в 10 раз** (с 500 МБ до 50 МБ).

## Автоматический конвейер (GitLab CI/CD)
В корне проекта настроен файл `.gitlab-ci.yml`, реализующий концепцию **Zero Downtime Deployment (Деплой без простоя)**:
1.  **Stage: build-and-pack** — Локальный `gitlab-runner` (Shell-исполнитель) ловит пуш кода, собирает Docker-образ и пушит его в закрытый **GitLab Container Registry**.
2.  **Stage: deploy-to-k8s** — Раннер динамически создает изолированную сессию безопасности SSH (права `600`), пробивает интернет, заходит на Мастер-ноду K8s и запускает плавное обновление (`rollout restart`). Kubernetes запускает новые Поды, проверяет их здоровье и только потом гасит старые.

```bash
# Получение списка пользователей
curl http://localhost:8080/users
```
